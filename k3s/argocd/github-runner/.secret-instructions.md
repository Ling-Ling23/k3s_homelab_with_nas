# GitHub Runner Secrets

⚠️ **IMPORTANT: DO NOT COMMIT secret.yaml TO GIT!**

This directory contains an example secret file. To create the actual secret:

## Steps:

1. Copy the example:
```bash
cp secret.yaml.example secret.yaml
```

2. Edit `secret.yaml` and replace `ghp_YOUR_GITHUB_PERSONAL_ACCESS_TOKEN_HERE` with your actual token

3. Apply to cluster:
```bash
kubectl apply -f secret.yaml
```

4. Delete the file (don't leave it around):
```bash
rm secret.yaml
```

Or apply directly without saving:
```bash
# Edit the token in the command below
cat <<EOF | kubectl apply -f -
apiVersion: v1
kind: Secret
metadata:
  name: github-runner-secret
  namespace: github-runner
type: Opaque
stringData:
  GITHUB_TOKEN: "ghp_YOUR_TOKEN_HERE"
  REPO_URL: "https://github.com/Ling-Ling23/k3s_homelab_with_nas"
EOF
```

## Better approach: Sealed Secrets

For GitOps, use Sealed Secrets (#6 in ROADMAP) to safely commit encrypted secrets to Git.

## Get GitHub Token:

1. Go to: https://github.com/settings/tokens
2. Generate new token (classic)
3. Select scopes: `repo` + `workflow`
4. Copy token and use in secret.yaml
